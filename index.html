01 — About

THE MISSION

100% Autonomous
5 AI Agents
4 Kill Chain Phases
APL2 Open Source

Let's be honest. The "AI + hacking" space is exhausting. Every other week, someone drops a demo: "Look, GPT can run nmap!" Cool. Then what? It either ends up as a party trick that no one actually uses in production — or worse, it crosses a line nobody should cross.

Another AI hacker? Not this one. Decepticon doesn't run nmap and write a report. It thinks in kill chains — reconnaissance, exploitation, privilege escalation, lateral movement, persistence — executing multi-stage operations the way a real adversary would.

Traditional red teaming demands hundreds of hours of manual work — scanning, enumerating, pivoting, documenting — most of it repetitive, all of it exhausting. Meanwhile, the attack surface grows faster than any human team can cover.

Autonomous Hacking changes the equation. AI agents handle the grind. The human sets the mission, defines the rules, and focuses on what machines still can't do — intuition, judgment, and creative thinking.

"Delegate the repetitive. Focus on the decisive."

// Operations Framework

RoE
Rules of Engagement

Defines scope, timing, and hard boundaries. Enforced every iteration by the orchestrator. Violate this and you're not a red teamer — you're a criminal.

ConOps
Concept of Operations

Threat actor profile, attack methodology, the "who are we pretending to be" document. Sets the adversary emulation context.

Deconf
Deconfliction Plan

Separates red team activity from real threats. Source IPs, user-agents, time windows, and a shared code for real-time deconfliction with the SOC.

OPPLAN
Operations Plan

Full mission plan. Objectives, kill chain phases, acceptance criteria. Every action maps to a purpose. Generated by Soundwave, fed to the orchestrator.

02 — Principles

WHY DECEPTICON?

Penetration testing finds vulnerabilities. Red teaming answers a harder question: can your organization survive a real attack? Four principles guide everything we build.

Real Red Teaming, Not Checkbox Security

Decepticon emulates actual adversary behavior — not just running CVE checks against a list of ports. It reads an operations plan, adapts to what it finds, and pursues objectives through whatever path opens up. The goal is to test your defenses the way they'll actually be tested.

adversary-emulation roe-enforced

Interactive Shell Sessions

Real offensive tools are interactive. sliver-client, msfconsole, evil-winrm. Decepticon runs every command inside persistent tmux sessions with automatic prompt detection. When a tool presents an interactive prompt, the agent detects it and sends follow-up commands — the same way a human operator would.

tmux-sessions prompt-detection parallel-shells

Complete Isolation

Every command runs inside a hardened Kali Linux sandbox on a dedicated operational network (sandbox-net), fully isolated from the management infrastructure (decepticon-net). No cross-network access. LangGraph reaches the sandbox exclusively via Docker socket — not the network.

kali-sandbox dual-network docker-socket

CLI-First

Security work belongs in the terminal. Decepticon's interface is a real-time streaming CLI built with Ink — no browser tabs, no dashboards, no context switching. You see what the agent sees, as it happens. Pure signal, zero noise.

ink-cli real-time-streaming terminal-native

C2 Integration (Sliver)

Sliver C2 team server runs on the operational network. mTLS, HTTPS, and DNS-based C2 channels. Implant generation, deployment, and session management. Post-exploitation via C2 sessions: credential harvesting, lateral movement, internal recon.

03 // FEATURE

MITRE ATT&CK Mapping

ATT&CK mapping at every layer — not added after the fact. Objectives carry technique IDs (T1190, T1003.001). Skills declare ATT&CK techniques in frontmatter. ConOps defines threat actor TTPs as ATT&CK IDs.

mitre-attack ttp-mapping threat-intel
🔀
04 // FEATURE

Multi-Model Routing

LiteLLM proxy routes to any backend (Anthropic, OpenAI, Google). Three profiles: eco, max, test. Each role has automatic fallback. Provider outage or rate limit → seamless switch. Zero downtime model transitions.

litellm multi-provider auto-fallback
📚
05 // FEATURE

Skill System

Progressive skill disclosure — only frontmatter loaded initially, full content on-demand. Skills organized by kill chain phase with MITRE ATT&CK tags. Covers OSINT, web exploitation, AD attacks, privilege escalation, lateral movement, credential access, defense evasion, and OPSEC.

progressive-disclosure kill-chain-phases
📋
06 // FEATURE

Engagement Planning

The Soundwave agent interviews the operator and generates a complete engagement package — RoE, ConOps, Deconfliction Plan, and OPPLAN. The OPPLAN feeds directly into the autonomous loop; the RoE is enforced every iteration.

05 — Installation

DEPLOY

Prerequisites: Docker + Docker Compose v2. That's it. No local Python, no Node.js, no dependency hell. Just containers.
Quick Install 
# One-line install — pulls everything
curl -fsSL https://raw.githubusercontent.com/PurpleAILAB/Decepticon/main/scripts/install.sh | bash
Configure & Launch 
# Set your Anthropic or OpenAI API key
decepticon config

# Start all services + open interactive CLI
decepticon
CLI Reference 
decepticon               # Start + open interactive CLI
decepticon demo          # Run guided demo (full kill chain)
decepticon config        # Edit API keys and settings
decepticon stop          # Stop all services
Try The Demo 
# Set your API key first (one-time)
decepticon config

# Run the full kill chain demo
# Launches Metasploitable 2 → port scan → vsftpd exploit
# → Sliver C2 implant → credential harvesting → internal recon
decepticon demo
Developer Setup (Contributors) 
git clone https://github.com/PurpleAILAB/Decepticon.git
cd Decepticon

# Python agents
uv sync --dev

# CLI client
cd clients/cli && npm install && cd ../..

# Build + run
docker compose up -d --build
cd clients/cli && npm run dev
